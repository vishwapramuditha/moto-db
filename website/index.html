Open Source & Git-Powered

A database built on modern developer principles. Zero friction, zero lock-in, and fully community-driven.

Architecture

Git-as-a-Database

Instead of hitting a heavy SQL or NoSQL database hosted on our servers, Moto-DB is stored as raw JSON files in Git. Your requests are served directly from GitHub via jsDelivr CDN edge locations, guaranteeing 100% uptime and zero cost to the community.

API Accessibility

Zero Signups or Rate Limits

No registration, no API keys, and no rate limits. Simply query data directly using standard AJAX fetch or curl calls. Perfect for prototyping, side projects, or high-performance client-side dashboard integrations.

Community Owned

Fully Open Contribution

The data scraper scripts run on automated GitHub Action schedules. If you spot a missing sprint race result, an incorrect lap record, or want to add a new motorsport series, open a Pull Request. Moto-DB is owned and audited by its users.

How Moto-DB Works

A simple, transparent, and resilient pipeline powering real-time motorsports schedules and results.

Step 1

Automated Scrapers

Python scraper scripts run automatically on scheduled intervals inside GitHub Actions. They fetch fresh schedule timelines and classifications directly from verified motorsport sources.

Step 2

Git Database Push

Crawled data is compiled, validated against our strict JSON schema formats, and committed directly back to the public repository. All updates are visible in the repository commit logs.

Step 3

Multi-CDN Delivery

The jsDelivr CDN automatically picks up the pushed files and caches them at the edge. Developers can call the files directly from our global endpoints with sub-50ms latency.

Developer API

Query clean, structured JSON data directly in your applications without authentication.

Formula 1

  • GET /data/f1/drivers.json
  • GET /data/f1/{year}/schedule.json
  • GET /data/f1/{year}/results_{round}.json

MotoGP

  • GET /data/motogp/drivers.json
  • GET /data/motogp/{year}/schedule.json
  • GET /data/motogp/{year}/{short_name}/motogp_RAC.json

NASCAR

  • GET /data/nascar/drivers.json
  • GET /data/nascar/{year}/schedule.json
  • GET /data/nascar/{year}/{series}/results_{id}.json

IndyCar

  • GET /data/indycar/{year}/schedule.json
  • GET /data/indycar/{year}/results_{id}.json

WEC

  • GET /data/wec/{year}/schedule.json

Formula E

  • GET /data/formula_e/{year}/schedule.json
