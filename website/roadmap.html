Project Roadmap

A detailed look at historical milestones and the future direction of the Moto-DB project.

Phase 1

Formula 1 Integration

Complete schedules, results, driver profiles, and track data. Originally powered by Ergast, now migrated to Jolpica for robust historical tracking.

Phase 2

MotoGP Integration

Complete season data including Sprint Races, Main Races, rider profiles, and circuit data pulled from official Pulse Live endpoints.

Phase 3

NASCAR & IndyCar

Full support for the top US motorsport series. NASCAR includes Cup, Xfinity, and Truck series. IndyCar utilizes ESPN's IRL scoreboard API.

Phase 4

WEC Schedules

World Endurance Championship timetables parsing JSON-LD schemas directly from fiawec.com.

Phase 5

Formula E

Complete schedules and race results via automated parsing of Wikipedia's highly structured season tables.

Phase 6

World Rally Championship (WRC)

Adding staging times, rally schedules, and driver profiles for the premier off-road racing series.

Automated Data Pipelines

How Moto-DB collects, updates, and serves motorsports schedule and results data in real-time.

Step 1

Python Scrapers

Our open-source Python scripts (located in /scripts) query various official and public API endpoints, HTML pages, and JSON-LD schemas to extract clean schedules, track distances, constructor teams, and race classifications.

Step 2

GitHub Actions

GitHub Workflows execute the Python scraping scripts periodically (every hour or daily, depending on active race events). If new data is crawled, the workflow formats it, validates the JSON schema, and commits the updates back to Git.

Step 3

Edge Delivery

Once pushed to the repository, the updated files are immediately available on the jsDelivr CDN. jsDelivr distributes the files globally using SSD-backed edge caches, ensuring developers retrieve fresh results within minutes of a race ending.

Shape the Future Roadmap

Have ideas for new data integrations, like adding rally splits, endurance lap times, or sports-car driver specs? You don't have to wait. Moto-DB is fully open-source, and we welcome code contributions to expand our scraped datasets.
